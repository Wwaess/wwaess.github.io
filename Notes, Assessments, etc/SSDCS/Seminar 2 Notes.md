# Seminar 2 Notes

## Waterfall model

- Requirements
- Design
- Development
- Test
- Deployment
- Maintenance

While superficially simple, once you're on a subsequent stage, it's really hard to move backwards, say because a design feature doesn't work but only discovered not to work during testing. 

## Agile model

This uses the same list, but _cycles_ through those steps - it's more flexible in this regard.
